# Deployment

Kemet is a static Astro site deployed from a **single branch: `main`**.

`main` holds both the Astro **source** (`src/`, config, data, scripts) and, at
the repository root, the **built website** (`index.html`, `_astro/`, `images/`,
…). The cPanel server is a plain checkout of `main`, so updating the live site
is always just `git pull origin main`. The site never has to build anything.

`public/.htaccess` ships with the build and blocks public access to the source
folders (`src/`, `scripts/`, …) and source files (`*.ts`, `*.astro`, …), so
visitors only ever see the finished website.

---

## Develop locally

```bash
npm install        # first time only
npm run dev        # local preview at http://localhost:4321
# ...edit content in src/data, src/pages, src/config, etc...
```

Edit the **source** under `src/`. Do not hand-edit the built `.html` files at
the repo root — they are regenerated by the publish step.

---

## Publish (build + commit + push)

When you want the changes to go live, run one command from the repo root:

```bash
./scripts/publish.sh                  # auto timestamped commit message
./scripts/publish.sh "release: v1.2"  # or a custom message
```

### What the script does

1. `npm run check` then `npm run build` (aborts on any error).
2. Removes root files a previous publish produced that the new build no longer
   emits (tracked in `.publish-manifest`).
3. Copies `dist/*` over the repo root.
4. Commits **everything pending** (source edits + rebuilt site) and pushes
   `main`.

If nothing changed since the last publish, it stops without committing.

---

## Update the live server

Because `main` already holds the finished HTML/CSS, the cPanel server never
installs Node or builds anything:

```bash
cd ~/public_html
git pull origin main
```

That's the entire server-side workflow.

### First-time server setup (once only)

If `public_html` is not yet a checkout of `main`:

```bash
cd ~/public_html
mkdir -p ~/public_html_backup_$(date +%F)
mv ~/public_html/* ~/public_html/.[!.]* ~/public_html_backup_$(date +%F)/ 2>/dev/null || true
git init
git remote add origin https://github.com/roza-tour/kemet.git
git fetch origin main
git checkout -b main origin/main
git reset --hard origin/main
```

Confirm `index.html` sits directly inside `public_html`. From then on, every
update is just `git pull origin main`.

> If a future `git pull` ever reports a conflict because a file was changed on
> the server, snap back to the exact published state with:
> `git fetch origin main && git reset --hard origin/main`

---

## Summary

```
Develop:   edit src/ on main   →  npm run dev
Publish:   ./scripts/publish.sh   (build + commit + push main)
Go live:   git pull origin main   (on the cPanel server)
```
